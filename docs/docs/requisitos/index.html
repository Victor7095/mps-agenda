Requisitos Cronoz

Este documento, visa apresentar os requsitos (funcionais e não funionais) do sistema assim como regras de negócios e casos de uso. Os requisitos de um sistema são descrições dos serviços fornecidos pelo sistema e as suas restrições operacionais. Esses requisitos refletem as necessidades dos clientes de um sistema que ajuda a resolver algum problema, por exemplo, controlar um dispositivo, enviar um pedido ou encontrar informações (SOMMERVILLE,2011).

3. Requisitos Não Funcionais

São restrições aos serviços ou funções oferecidos pelo sistema.

#
Descrição
Métrica
RNF001
Internet
O aplicativo só poderá ser utilizado mediante conexão via internet.
RNF002
Versão Android
O aplicativo só funcionará na versão Android 4 ou superior.
RNF003
Versão IOS
O aplicativo só poderá ser executado na versão IOS 12 ou superior
RNF004
Tamanho de imagem
O aplicativo será exibido normalmente em todas as resoluções de telas devido a responsividade.
RNF005
Desempenho
As ações do marketplace serão executadas em menos de 2 segundos.
RNF006
Trafégo
A aplicação suportará até 2000 acessos simultâneos
RNF007
Navegadores
A versão para web funcionará em todos os navegadores mais utilizados
RNF008
Segurança
O sistema garantirá a integridade dos dados dos usuários. (HTTPS, Captcha)
